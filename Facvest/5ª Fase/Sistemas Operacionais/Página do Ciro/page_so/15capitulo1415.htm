14.2 OBJETOS E CAPACIDADES NO AMOEBA

0 conceito básico de todos os servidores Amoeba e dos serviços prestados por tais servidores é o conceito de objeto Um objeto é um conjunto de dados encapsulados, sobre os quais são realizadas certas operações muitas bem definidas. Um objeto é. em sua essência, um tipo abstrato de dados. Os objetos são passivos, não tendo nenhuma entidade capaz de “fazer” que coisas aconteçam. Ao contrário, cada objeto é gerenciado por um processo servidor.
Para realizar uma operação sobre determinado objeto, o cliente deve executar uma chamada remota a procedimento para um servidor, especificando o objeto, a operação a ser realizada, e, caso queira, alguns parâmetros que julgue necessários. 0 servidor faz o trabalho requisitado e retoma a resposta ao cliente. As operações são realizadas de forma síncrona ou seja. após iniciar uma chamada remota a um servidor, solicitando algum trabalho, a linha de controle do cliente é bloqueada até que o servidor responda. Outras linhas de controle, entretanto, no mesmo processo poderão estar rodando.
