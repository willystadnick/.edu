Também se pode utilizar a expiração, onde uma fatia de tempo T é dada a cada chamada remota, se o trabalho não terminar é solicitado um novo quantum de tempo. Se após uma queda o servidor esperar por um tempo T, terá certeza de que todos os órfãos foram eliminados. O problema é a escolha de um valor para o tempo T, pois as chamadas podem requisitar desde tarefas simples, que consomem pouco tempo, até as mais complexas, que gastam muito tempo para serem realizadas.

O sucesso ou fracasso de um sistema distribuído depende de sua performance, que depende da velocidade de comunicação entre suas partes componentes.

PROTOCOLOS PARA CHAMADA REMOTA A PROCEDIMENTO

A escolha do protocolo deve ser analisada, teoricamente qualquer um vai servir, desde que transporte os bits do kernel do cliente para o do servidor. A primeira decisão diz respeito ao uso de protocolos orientados a conexão ou que não requerem conexão explícita entre as partes comunicantes. No primeiro caso o cliente se liga ao servidor, estabelecendo uma conexão explícita, por onde passa todo o tráfego.
