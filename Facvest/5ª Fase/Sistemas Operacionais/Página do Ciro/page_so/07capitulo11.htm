Implementação

Deve ficar claro desde já que, se cada processo que executar uma transação simplesmente atualizar os objetos que ele usar (arquivos, registros de bases de dados etc.), então as transações não serão atômicas e as mudanças não vão evaporar-se instantaneamente no caso da transação abortar. Além do mais, os resultados da execução de várias transações tampouco serão serializados. Fica clara a necessidade de alguma outra maneira de implementar as transações. Existem dois métodos que são comumente usados na implementação das transações. Ambos serão discutidos a seguir.

Área de Trabalho Privada

Conceitualmente, quando um processo inicia uma transação, ele recebe uma área de trabalho privada, contendo todos os arquivos e demais objetos aos quais ele tem acesso. Até que a transação se complete ou aborte. Tudo o que o processo ler ou escrever vai para área de trabalho privada, em vez de para a área “real”, como estamos denominando o sistema de arquivos normal do sistema.
O problema com esta técnica é que o custo de copiar tudo para área privada é proibitivo. No entanto, podem ser feitas algumas otimizações para viabilizar o esquema proposto. A primeira delas é baseada na constatação de que, quando um processo lê um arquivo mas não o modifica, não há necessidade de uma cópia privada. Conseqüentemente, quando um processo inicia uma transação, basta criar para ele uma área de trabalho privada vazia, exceto por um ponteiro para área de trabalho-mãe.
