3.13.2 Capacidades

Cada capacidade possui um campo de tipo que informa qual o tipo de objeto ela é, em campo de direitos, que é um mapa de bits indicando as operações permitidas, e um campo de objeto, que é um ponteiro para o próprio objeto. As listas de capacidade são elas próprias objetos e podem ser apontadas a partir de outras listas de capacidades, desta forma facilitando o compartilhamento de subdomínios.

3.13.3 Modelos de Proteção

Harrison et al (1976) identificaram seis operações primitivas sobre a matriz de proteção que podem ser usadas como base na modelagem de qualquer sistema de proteção. Tais operações são: criar objeto, apagar objeto, criar domínio, apagar domínio, inserir direito e remover direito.
Estas seis primitivas podem ser combinadas em comandos de proteção. São estes comandos de proteção que os programas de usuário podem executar para mudarem a matriz. Eles não podem executar as primitivas corretamente. Por exemplo, o sistema pode ter um comando para criar um novo arquivo, o qual deve fazer um teste antes de criá-lo, a fim de verificar se o arquivo já existe. Se não existir, ele deve criar um novo objeto, e dar ao proprietário todos os direitos sobre ele. Deve também haver um comando para permitir que o proprietário de permissão a quem quer que seja para ler o arquivo, inserindo o direito de ‘leitura’ na nova entrada do arquivo em cada um dos domínios.
