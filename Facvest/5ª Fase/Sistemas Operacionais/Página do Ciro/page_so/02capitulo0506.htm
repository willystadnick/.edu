Dispositivos de caracter: liberam ou aceitam um conjunto de caracteres, sem seguir uma estrutura de bloco e não é endereçável, como exemplo podem ser citados os terminais, mouses ou qualquer outro dispositivo quem não se ajustam às características de um disco. É bom exemplificar que esta divisão não é perfeita, pois existem dispositivos que não se ajustam a eles como é o caso das telas de micros ou terminais mapeados na memória. Mas é um modelo suficiente para quem busca a independência de dispositivos. O sistema operacional faz entrada/saída, escrevendo comandos nos registradores das controladoras, estes comandos possuem parâmetros que também precisam ser carregados nos registradores. Como exemplo o FORMATE na unidade de disquete, quando este comando é aceito o processador deixa a controladora trabalhando sozinha, quando a execução termina, a controladora gera uma interrupção, de forma a permitir que o sistema operacional ganhe o processador, verificando o resultado da operação e o estado do dispositivo lendo as informações contidas nos registradores das controladoras.

5.1.3 ACESSO DIRETO À MEMÓRIA (DMA)

As controladoras que suportam operações de acesso direto à memória (DMA) são as desenvolvidas para dispositivos de blocos. Para melhor compreensão vamos exemplificar a leitura de disco. A controladora lê o bloco até que esteja todo em seu buffer interno, calcula a ocorrência de erros, depois a controladora gera uma interrupção. O sistema operacional lê o bloco de disco escrito no buffer da controladora, bit a bit ou palavra por palavra (loop) e armazena na memória.
