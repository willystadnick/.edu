3.4.4 Nós-i

O quarto e último método para controlar a correspondência entre blocos no disco e arquivos consiste em associar a cada arquivo de uma pequena tabela denominada nó-i (nó índice), que lista os atributos e os endereços em disco dos blocos do arquivo.
Os primeiros endereços de disco são armazenados no próprio nó-i, de maneira que, para arquivos pequenos, toda a informação necessária é encontrada diretamente no nó, sendo transferida do disco para memória quando o arquivo for aberto. Em arquivos maiores, um dos endereços no nó-i, é o endereço de um bloco do disco chamado bloco indireto simples. Tal bloco contém endereços adicionais no disco relativos ao arquivo em questão. Se o mesmo assim houver necessidade de mais endereços, devido ao tamanho do arquivo, outro endereço no nó-i, denominado bloco indireto duplo, contém o endereço do bloco que possui uma lista de blocos de endereço simples, relativos a este arquivo. Cada um destes blocos indiretos simples aponta para algumas centenas de blocos de dados. Se os blocos duplos não forem suficientes, pode-se usar blocos indiretos triplos. O UNIX segue este esquema.

3.5. Implementação de Diretórios

Antes que um arquivo possa ser lido, ele precisa ser aberto. Ao abrir uma arquivo, o sistema operacional usa o nome do caminho fornecido pelo usuário para localizar sua entrada no diretório.
