Capitulo 4

INTRODUÇÃO

As aplicações computacionais, quaisquer que sejam, precisam armazenar e recuperar informação. Enquanto um processo executa, ele é capaz de armazenar certa quantidade de informação em seu próprio espaço de endereçamento. Para algumas aplicações este espaço é suficiente, porém para outras ele é muito pequeno.
Outro problema existente no armazenamento dos dados, é o fato de a informação ser perdida quando o processo termina. A perda dessas informações não é aceitável, já que para algumas aplicações é necessário que as informações sejam mantidas por tempo indeterminado.
Pode-se citar também, a necessidade de que vários processos tenham acesso ao mesmo conjunto de informações ao mesmo tempo.
A solução para os três problemas apontados a cima é manter a informação armazenada em disco ou em qualquer outro dispositivo externo de armazenamento, em unidades denominadas arquivos.
Os arquivos são gerenciados pelo sistema operacional. A parte do sistema operacional responsável pelo tratamento dos arquivos é denominada sistema de arquivos, que é o objeto de estudo deste texto. No conteúdo que segue encontram-se as formas de estruturá-los, acessá-los, utilizá-los, protegê-los e implementá-los; a apresentação dos arquivos, de que são constituídos e como são identificados. Abordamos ainda, a interface dos arquivos e dos diretórios, de uma forma detalhada, com o objetivo de proporcionar o entendimento e a capacitação para a resolução dos problemas acima mencionados.
