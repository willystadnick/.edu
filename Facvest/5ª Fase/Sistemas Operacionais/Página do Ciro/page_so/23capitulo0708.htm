2.4. AS CHAMADAS DE SISTEMA DO UNIX

2.4.1. Chamadas de Sistema para Gerência de Processos no UNIX

O primeiro grupo de chamadas trata da gerência dos processos. O uso da fork é a única maneira de se criar um novo processo. Ela cria uma cópia exata do processo original, incluindo todos os seus descritores de arquivo, registradores e tudo mais. Após a execução de uma fork, tanto o processo original quanto sua cópia (o pai e o filho) seguem cada um seu caminho. Todas as variáveis têm valores idênticos no momento da execução da fork, mas, uma vez que a imagem completa do pai é copiada para dar origem ao filho, as mudanças subseqüentes em um deles não vão afetar o outro. A chamada fork retorna um valor, que é 0 para o filho, e igual ao identificador do processo-filho, pid, para o pai. Usando o pid devolvido pelo sistema, ambos os processos podem saber quem é o pai e quem é o filho.
No caso do Shell, o processo-filho precisa executar o comando digitado pelo usuário. Ele faz isto usando a chamada EXEC, que faz com que toda sua imagem de memória seja substituída pelo arquivo especificado em seu primeiro parâmetro.
A EXEC tem três parâmetros: o nome do arquivo a ser executado, um ponteiro para o array de argumentos, e um ponteiro para o array do ambiente.
A chamada EXEC é efetivamente a mais complexa de todas as chamadas de sistema. As demais são muito mais simples. Uma bem simples, a EXIT que todos os processos devem chamar ao término de suas execuções e ela só possui um parâmetro.
Existem várias chamadas de sistema relativa a sinais, as quais são usadas de formas variadas.
