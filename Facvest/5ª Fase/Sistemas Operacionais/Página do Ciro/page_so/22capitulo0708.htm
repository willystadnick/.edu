Os arquivos especiais podem ser acessados exatamente da mesma forma que os arquivos comuns. As chamadas read e write vão funcionar muito bem neste caso.
O UNIX fornece um programa chamado lpr para permitir que os usuários imprimam seus arquivos. O programa lpr simplesmente copia os arquivos especificados em seus argumentos para um diretório de spool, de onde um processo especial, com acesso a /dev/lp, retira e imprime tais arquivos em ordem. Deve ficar claro que /dev/lp não precisa necessariamente pertencer ao superusuário. Basta que o processo especial tenha acesso a tal arquivo. O fato de se ter os dispositivos de entrada/saída integrados ao sistema de arquivo dá uma imensa flexibilidade no acesso às informações.
O UNIX, em vez de fazer uma escolha e obrigar a todos a seguir tal escolha, permite que todas as funções especiais, sejam customizadas pelo próprio usuário. Existe uma chamada de sistema especial para realizar estas customizações. Esta chamada também manipula a tabulação, habilita e desabilita o eco, converte o retorno do carro em alimentação de linha, além de outros itens similares. Sua utilização em arquivos especiais de blocos ou em arquivos comuns não é permitida.
A entrada/saída envolvendo redes locais, cujo desenvolvimento pioneiro se deu no UNIX de Berkeley. O conceito central do projeto de Berkeley é o do soquete. Os soquetes são similares às caixas postais e aos aparelhos telefônicos, à medida que permitem que os usuários tenham uma interface adequada com a rede, da mesma forma que as caixas postais permitem que as pessoas tenham uma interface com o sistema postal, e os aparelhos telefônicos fornecem uma interface adequada das pessoas com o sistema de telefonia de determinada localidade. Os soquetes podem ser criados e destruídos dinamicamente.
