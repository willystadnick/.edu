Uma outra propriedade importante do sistema de arquivos do UNIX é o bloqueio. Em algumas aplicações, dois ou mais processos podem estar usando o mesmo arquivo ao mesmo tempo, o que pode possibilitar a ocorrência das indesejadas condições de corrida. Uma solução para evitá-las é programando a aplicação usando regiões críticas. No entanto, se os processos pertencerem a usuários diferentes, que nem mesmo se conhecem, este tipo de coordenação torna-se muito difícil para não dizer impossível de ser obtido.
Quando da colocação de um bloqueio, o processo deve especificar se ele deseja ser suspenso na eventualidade de o bloqueio não se efetivar por qualquer motivo. Se ele optar por ser suspenso, quando o bloqueio for removido, o processo é acordado e o novo bloqueio é efetivado. Se o processo escolher não ser suspenso quando seu bloqueio não puder ser efetivado, a chamada de sistema retorna imediatamente, com o código informando se o bloqueio teve ou não sucesso.

2.3.4. Entrada/Saída no UNIX

No UNIX tem conectado a ele dispositivos de entrada/saída tais como terminais, discos, impressoras e redes. É necessário que seja definido algum esquema que permita que programas venham a acessar tais dispositivos. Apesar de serem possíveis diversos tipos de soluções, a solução do UNIX é integrar todos os dispositivos no sistema de arquivo, nos chamados arquivos especiais. Cada dispositivo de entrada/saída recebe um nome de caminho, usualmente no diretório /dev.
