A idéia é associar a cada senha um número randômico de n bits. Tal número muda toda vez que a senha é trocada e é armazenado no próprio arquivo de senhas, de forma não-criptografada, de maneira que qualquer pode lê-los se assim o desejar. Em vez de simplesmente armazenar a senha criptografada no arquivo de senhas, a senha e o número randômico são concatenados e depois criptografados juntos. O resultado da criptografia é então armazenado no arquivo das senhas.

3.12.2 Identificação Física

Uma forma de autenticação consiste em medir determinadas características físicas dos usuários, muito difíceis de serem forjadas. Por exemplo, pode ser instalado em um terminal um dispositivo para verificar as impressões digitais ou para reconhecer a voz dos usuários. A busca torna-se prática se o usuário informar ao terminal quem ele é, em vez de fazer com que o sistema seja obrigado a pesquisar toda a base de dados. O reconhecimento visual direto do usuário ainda não é possível, embora tudo leve a crer que será num futuro próximo.

3.13 Dominios De Proteçao

Um sistema de computador contem muitos objetos aos quais é necessário dar proteção. Tais objetos podem ser hardware, tais como processadores, segmentos de memória, terminais, unidades de disco, impressoras, ou podem ser software, tais como processos, arquivos, bases de dados ou semáforos.
Cada objeto tem um único nome, através do qual ele é referenciado, e um conjunto de operações que podem ser realizadas sobre ele. As operações de READ e WRITE são apropriadas para atuar sobre arquivos, assim como as de UP e DOWN só terão significado se aplicadas sobre semáforos.
