Em tais sistemas, cada arquivo pode ser mapeado para um segmento próprio, de forma que o byte k do arquivo corresponda ao byte k do segmento. Apesar do mapeamento de processos eliminar a necessidade de entrada/saída, tornando a programação mais simples, esta técnica introduz alguns pequenos problemas. Em primeiro lugar, é difícil para o sistema saber o tamanho exato do arquivo de saída. É fácil informar o número de página de mais alta ordem que foi escrita, mas não há como saber quantos bytes desta página foram escritos.
Um segundo problema ocorre (potencialmente) se o arquivo for mapeado por um processo e aberto por outro para fazer uma leitura convencional. Se o primeiro processo modificar uma página, esta modificação não vai se refletir no arquivo em disco, até que a pagina modificada seja removida da memória pelo algoritmo de substituição. O sistema deve tomar muito cuidado para ter certeza de que estes dois processos não estão tratando com duas versões inconsistentes do arquivo.
Um terceiro problema com o mapeamento é que o arquivo pode ser menor do que o segmento, ou mesmo maior do que todo o espaço de endereçamento virtual. A única maneira de se resolver isto é programando a chamada MAP de modo a fazê-la mapear um pedaço do arquivo, e não o arquivo inteiro. Apesar de funcionar, esta solução é bem menos conveniente do que o mapeamento de todo o arquivo.
