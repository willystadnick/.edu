3.17. IMPLEMENTAÇÃO DE ENTRADA/SAÍDA NO MS DOS

As operações realizadas no MS DOS acontecem através de arquivos especiais e caracteres. Esses arquivos são ligados a dispositivos que manipulam um carater por vez.
A cada arquivo especial esta um drive que contem o código que realiza a operação de entrada /saída .
A idéia por traz dos drives de dispositivos é de mostrar o MS DOS com uma interfase padrão a todos os dispositivos do hardware.Quando arquivos especiais são lidos ou escritos por um programa, incluindo todos os acessos ao sistema de arquivos, o MS DOS chama o drive correspondente numa forma padronizada e informa a ele aquilo que deseja obter .Esse tipo de coisa desacopla o sistema operacional dos detalhes do hardware.
Um drive só pode ser usado para um arquivo de caracteres ou de blocos, nunca para os dois, pois suas interfases são diferentes, e suas funções também não são as mesmas. Cada drive é um programa separado, e compilado para um arquivo .com ou . exe.
Sua estrutura é basicamente:
1º Cabeçalho de 18 bytes basicamente;
2º Código propriamente, composto de duas partes uma para aceitar a solicitação do MS DOS a outra para aceitar a saída.
Quando o drive é instalado ele vai para uma espécie de pilha de drives, o MS DOS mantém um ponteiro para o inicio desta pilha em uma variável interna, e assim pesquisa a pilha, sendo assim os drives instalados por ultimo tem procedência sobre os outros.
Uma pessoa que deseja usar a seqüência de escape do código Ansi , deve colocar c:\bin\Ansi.sys (isso faz com que carregue para com a partir do arquivo\bin\ansi.sys armazenando no disco rígido).
