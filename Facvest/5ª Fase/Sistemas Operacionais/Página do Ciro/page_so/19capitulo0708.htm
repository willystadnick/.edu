O segmento de código contém as instruções de máquina que formam o código executável do programa. Ele é produzido pelo compilador e pelo montador, que traduzem os programas em C, Pascal, FORTRAN, ou em qualquer outra linguagem, para o código de máquina.
O segmento de dados armazena as variáveis do programa, seus strings, arrays e outros dados necessários à execução do programa. Ele contém duas partes, a dos dados inicializados e a dos dados não-inicialiazados.
A parte inicializada do segmento de dados contém as variáveis e as constantes do compilador que precisam ter um valor inicial quando é iniciada a execução do programa. A existência de dados não-inicializados é na realidade apenas uma otimização. Quando uma variável global não for explicitamente inicializada, a semântica da linguagem C faz com que ela assuma o valor 0.
O segmento de pilha, em muitas máquinas, ele começa no topo do espaço de endereçamento virtual e cresce em direção ao endereço 0. Se a pilha tentar crescer além do espaço reservado ao segmento, será gerada uma interrupção, fazendo com que o sistema operacional abaixe um pouco mais o limite inferior da pilha, aumentando-a.
Os segmentos de dados e de pilha não podem nunca ser compartilhados. Se um dos dois precisar crescer e não houver espaço adjacente para permitir o crescimento, o segmento deve ser transferido para algum outro lugar da memória, com espaço suficiente.
