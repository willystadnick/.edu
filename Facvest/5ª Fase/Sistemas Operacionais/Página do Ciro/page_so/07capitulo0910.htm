A ARQUITETURA DO NFS

A característica básica da arquitetura do NFS é o fato de os servidores exportarem seus diretórios e de os clientes montarem remotamente estes diretórios. Se dois ou mais clientes montarem o mesmo diretório ao mesmo tempo. Eles poderão comunicar-se através do compartilhamento de arquivos de seus diretórios comuns. Um programa em um cliente pode criar um arquivo, e outro programa em um cliente diferente pode ler este arquivo. Feitas as montagens devidas, não é preciso fazer mais nada para permitir o compartilhamento dos arquivos. Os arquivos compartilhados são justamente aqueles pertencentes a hierarquia de diretóris de várias máquinas, podendo ser lidos e escritos da maneira usual. Esta simplicidade é, o maior atrativo do NFS.

OS PROTOCOLOS DO NFS

O NFS usa o mecanismo de proteção UNIX, com os bits rwx para o proprietário para o grupo, e para os demais usuários. Originalmente, cada mensagem de solicitação ao servidor continha apenas as identificações do usuário e do grupo do processo que chamou, informações necessárias para que o servidor possa validar o acesso. Na verdade, o sistema acredita que o cliente nunca venha blefar. Ocorre que vários anos de experiência demonstraram que este comportamento é no mínimo ingênuo. Atualmente, pode ser usado um sistema público de criptografia para estabelecer uma forma segura de validar o cliente e o servidor a cada solicitação e a cada resposta.
