14.2.2. Implementação do Servidor de Diretório

O servidor é um componente crítico do sistema Amoeba, tendo sido implementado para ser tolerante a falhas. Sua estrutura básica de dados é uma matriz de pares de capacidade, armazenada em uma partição privada do disco.
Quando se cria um dir, o número do objeto colocado em sua capacidade é um índice para essa matriz. Ao se apresentar uma capacidade de dir, o servidor inspeciona o número do objeto que ela contém, e o utiliza para buscar o par de capacidade correspondente nessa matriz.
Quando se modifica um dir, cria-se-lhe um novo arquivo-bala, e as matrizes nas partições privadas do disco são modificadas. A segunda cópia é criada mais tarde por uma linha de controle rodando em background. Destroem-se então, os diretórios antigos. Embora esse mecanismo apresente um overhead extra, ele fornece um grau de confiabilidade muito maior que o dos sistemas de arquivo tradicionais.

14.6.3. Servidor de Replicação

Os objetos gerenciados pelo servidor de diretório podem ser replicados automaticamente através do servidor de replicação, que realizam a chamada replicação perigosa, que significa que, ao se criar um arquivo ou outro objeto qualquer, só é feita uma única cópia. Depois disso, o servidor de replicação pode ser chamado, podendo produzir tantas réplicas quantas forem necessárias. Esse servidor, em vez de ser chamado diretamente, é mantido rodando em background durante todo o tempo, observando, a intervalos, partes específicas do servidor de dir.
