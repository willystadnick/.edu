Uma variável mutex é idêntica a um semáforo binário, podendo assumir somente dois estados, bloqueados ou desbloqueados. A tentativa de bloquear uma variável mutex desbloqueada faz que ela passe ao estado de bloqueada. A linha que efetuar a primitiva continua sua execução. A tentativa de bloquear uma variável mutex que esteja bloqueada obriga a linha de controle que fez a tentativa a ficar bloqueada até que outra linha desbloqueie a variável mutex. Se mais de uma linha de controle estiver esperando nessa variável mutex, somente uma delas será ativada na ocasião de desbloqueio da variável. Alem das chamadas para bloquear e desbloquear as variáveis mutex, existe uma chamada que tenta bloquear uma mutex, mas se não for capaz de faze-lo dentro de determinado intervalo de tempo, haverá geração de um código de erro.
A terceira forma de comunicação entre linhas de controle se dá através dos semáforos, que são mais lentos que as variáveis mutex, mas que, em certas ocasiões, são necessários. Funcionam como já se viu antes, a não ser quanto à existência de uma chamada adicional, fornecida para permitir que uma operação DOWN seja temporizada, caso não tenha sucesso em determinado intervalo de tempo.
Todas as linhas de controle são gerenciadas pelo kernel, resultando dessa metodologia de projeta a vantagem de, quando uma linha de controle executa uma chamada remota a procedimento, o kernel pode bloqueá-la a escalonar uma outra dentro do próprio processo, caso haja alguma pronta para execução. 0 escalonamento das linhas de controle se faz usando prioridades, tendo as linhas do kernel sempre prioridade maior que as linhas de controle de usuários.
