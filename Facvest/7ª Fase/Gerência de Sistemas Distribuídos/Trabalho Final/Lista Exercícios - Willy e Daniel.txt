Para levantar um serviço de rede usa-se o comando "service nome_do_serviço start". Existe algum procedimento usado para carregar automaticamente todos os serviços de rede? Como este procedimento é realizado?

Basta ativar os serviços de rede desejados criando um link no diretorio que se deseja apontando para o script correspondente em "/etc/rc.d/init.d" usando o comando "ln-s /etc/rc.d/init.d /etc/rc.d/rc3.d/S<script de serviço de rede desejado>".

Os serviços de rede podem ser instalados e atualizados através dos CD's de instalação do CL10 (3 CD's). ou seja 3 CD's para cada aluno. Isto seria tedioso e praticamente inviável. Qual alternativa isentaria o professor de reprozir dúzias de CD's? Observe que esta alternativa deve ser efetiva.

A alternativa efetiva seria configurar (editar) a lista de localização dos repositorios de arquivos disponiveis na Internet e/ou rede. A configuração se daria em comentar as linhas referentes ao CD de instalação e descomentar as linhas referentes as URL's apropriadas.O arquivo referente a esta configuração pode ser editado no /etc/apt/sources.list.

Quais os procedimento usados para incluir uma máquina numa rede pré-existente. Observe que esta máquina não possui nenhum utilitário facilitar para esta tarefa, ou seja, não existe nenhum serviço tal como o linuxconf instalado. Como será feita a atribuição de IP's: estático ou dinâmico?

Uma vez conhecido o endereço IP de um dos elementos da rede pode-se tentar aleatoriamente atribuir um IP contido na mesma faixa do elemento conhecido, utilizando o comando: pico /etc/sysconfig/network-script/ifcfg-eth0. Tambem pode-se editar o arquivo /etc/sysconfig/network, atribuindo o valor aleatorio na linha iniciada por "IPADDR=<IP>" dentro da faixa de IP's. Deve-se tambem editar os demais campos do arquivo afim de deixa-los com a mesma configuração do elemento de rede conhecido.

Caso seja uma rede que possua um servidor DHCP basta setar para "yes" onde está a linha "bootproto=none" no arquivo etc/sysconfig/network-script/ifcfg-eth0 e deixar os demais campos em branco.

Sobre o FTP:

a) Para que serve o FTP e em que situações um usuário o utilizaria? 

File Transfer Protocol – FTP é utilizado para viabilizar o transporte de arquivos de um host para outro. Ele é utilizado em situações em que o usuario necessita compartilhar arquivos que está contido em outro host que esteja conectado a sua rede.


b) Como o FTP é usado no servidor e no cliente? 

No servidor, o serviço FTP deve estar instalado e rodando para que o serviço possa ser requisitado pelos clientes que o fazem atraves do comando "ftp <ipservidor>" e realizam as tarefas desejadas.

c) Configurar o FTP para usuário aluno. Testar através de máquinas linux (browser e konqueror) e windows (browser). Observe que o FTP funciona através do konqueror com o seguinte comando ftp://usuario@IP. E também através do browser linux/windows com o seguinte comando ftp://usuario@IP.

Com o serviço FTP devidamente instalado e funcionando no servidor, deve-se então criar o usuário "aluno" e para ele definir uma senha, para que possa ser realizado a autenticação pelo serviço de FTP. O usuário, a partir de uma máquina cliente rodando o Linux, utiliza-se do comando "ftp <ipservidor>" informando login e senha previamente setados e realiza o acesso ao conteúdo disponibilizado no servidor FTP através do console/terminal. Para realizar o acesso através do browser ou gerenciador de arquivos e pastas (konqueror) basta digitar o comando "ftp://aluno@ipservidor" na barra de comandos do aplicativo. É fato que o mesmo procedimento se aplica ao browser do windows (IE).

d) Configure o servidor FTP para permitir entre 1 e 10 conexões. Testar as funcionalidades.

Para permitir tal acesso é necessario editar o arquivo "/etc/proftpd.conf" e definir o parametro de numero máximo de clientes (MaxClients) para 10. Para testar, é necessario que 10 clientes já estejam realizando acesso ao servidor, e nesse momento um 11 cliente tente efetuar conexão e então seja exibida uma mensagem de limite máximo de conexões atingido.

e) Configurar o FTP para ser acessado via usuário anonymous.
